# 🌿 苔玉养护日记

一个纯前端的苔玉、青苔盆景养护管理工具，帮助你记录每一颗苔玉的生长节奏，再也不会忘记谁该浇水了。

## ✨ 功能特性

### 核心功能
- **苔玉档案管理**：为每一颗苔玉建立独立档案，记录苔种、基质配比、摆放位置、备注等信息
- **养护记录追踪**：按时间线追加养护动作（浸水、喷雾、修剪、移动、观察等），支持上传状态照片缩略图
- **智能筛选排序**：按苔种、摆放环境、距上次浇水天数筛选，默认按浇水 urgency 排序，一眼看出哪几颗该浸水了
- **数据本地存储**：所有数据保存在浏览器 localStorage 中，无需后端服务
- **JSON 导入导出**：一键导出 JSON 备份文件，换设备时轻松迁移数据

### 特色设计
- **浇水提醒可视化**：每张苔玉卡片显示距上次浇水天数，并用颜色标识 urgency
  - 🟢 绿色（0-3 天）：状态良好
  - 🟡 黄色（4-6 天）：可以准备浇水
  - 🔴 红色（≥7 天）：急需浸水
- **照片压缩存储**：上传的照片自动压缩为 300px 缩略图，节省存储空间
- **响应式布局**：支持桌面和移动设备
- **模块化架构**：代码按功能拆分，方便后续扩展

## 🚀 本地启动

### 环境要求
- Node.js >= 16.0.0

### 模块说明

- **KokedamaList.vue**：负责展示苔玉卡片列表，提供筛选（苔种、位置、浇水天数范围）和排序功能
- **KokedamaForm.vue**：新增或编辑苔玉档案的弹窗表单，支持已输入过的苔种/位置自动补全
- **KokedamaDetail.vue**：展示单颗苔玉的完整信息，管理养护记录的增删，支持照片上传和预览
- **stores/kokedama.js**：使用 Pinia 管理所有苔玉数据状态，包括筛选排序逻辑、数据持久化
- **utils/storage.js**：封装 localStorage 的读写操作，以及 JSON 文件的导入导出功能
- **utils/date.js**：日期格式化、计算距今天数、生成唯一 ID 等工具函数

## 🔄 数据备份与迁移

### 导出数据
1. 点击页面右上角的「📤 导出数据」按钮
2. 浏览器会自动下载一个 JSON 文件，文件名格式为 `kokedama-backup-YYYY-MM-DD.json`
3. 建议定期导出备份，防止浏览器缓存清理导致数据丢失

### 导入数据
1. 点击页面右上角的「📥 导入数据」按钮
2. 选择之前导出的 JSON 文件
3. 系统会提示选择「覆盖现有数据」或「追加新数据」：
   - **覆盖**：删除所有现有数据，用导入的数据完全替代
   - **追加**：保留现有数据，只导入 ID 不重复的新记录

## 🔮 未来扩展方向

代码采用模块化设计，方便后续添加功能：

- **浇水周期推算**：根据每颗苔玉的历史浇水记录，自动推算最佳浇水周期
- **按苔种统计**：统计不同苔种的成活率、平均浇水间隔等数据
- **养护提醒通知**：浏览器通知提醒需要浇水的苔玉
- **数据可视化**：用图表展示浇水频率、状态变化趋势
- **多设备同步**：可选接入云端同步服务

## ⚠️ 注意事项

1. **数据仅存在本地**：请定期导出 JSON 备份，清理浏览器缓存或更换设备前务必导出数据
2. **照片存储限制**：照片以 base64 格式存储在 localStorage 中（约 5MB 上限），建议只上传必要的状态照片
3. **浏览器兼容性**：建议使用现代浏览器（Chrome、Firefox、Safari、Edge）
